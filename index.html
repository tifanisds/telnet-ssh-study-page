// 01 — FUNDAMENTOS

O que é uma Rede de Computadores?

Segundo Andrew S. Tanenbaum, uma rede de computadores é um conjunto de computadores independentes interconectados por uma única tecnologia. O principal objetivo dessa estrutura é facilitar o compartilhamento de recursos e informações, independentemente da distância física entre as máquinas.

As redes de computadores permitem que usuários em diferentes locações trabalhem colaborativamente, compartilhem arquivos, acessem servidores remotos e se comuniquem em tempo real. Para que essa comunicação funcione adequadamente, existem diversos protocolos de rede, cada um com um propósito específico.

Arquitetura Cliente-Servidor

A maioria das redes segue o modelo cliente-servidor, onde clientes solicitam recursos e servidores os fornecem. Essa arquitetura permite organização eficiente e controle centralizado dos dados.

Protocolo TCP/IP

A base de todas as redes modernas é o protocolo TCP/IP, que define como os dados são roteados e entregues entre dispositivos, garantindo confiabilidade e compatibilidade global.

Camadas de Protocolo

Os protocolos são organizados em camadas (aplicação, transporte, rede, enlace), cada uma responsável por diferentes aspectos da comunicação entre dispositivos.

// 03 — PROTOCOLOS

Protocolos de Rede

Conheça os principais protocolos utilizados em redes de computadores. Cada um deles desempenha um papel específico na comunicação entre dispositivos, desde o transporte de emails até a administração remota segura de servidores.

Protocolo

Telnet

Telecommunications Network Protocol, protocolo de acesso remoto que permite controlar computadores remotos via linha de comando. Transmite dados sem criptografia e é considerado obsoleto.

Porta padrão: 23

Função: Acesso remoto (inseguro)

Protocolo

SSH

Secure Shell — protocolo seguro para acesso remoto e administração de sistemas. Utiliza criptografia de ponta a ponta, garantindo que os dados transmitidos não possam ser interceptados.

Porta padrão: 22

Função: Acesso remoto seguro

Ver guia completo →
Protocolo

SMTP

Simple Mail Transfer Protocol — protocolo utilizado para o envio de emails através de redes. Trabalha na porta 25, 465 ou 587, permitindo que clientes de email entreguem mensagens a servidores de correio.

Porta padrão: 25, 465, 587

Função: Envio de emails

Ver guia completo →
Protocolo

POP3

Post Office Protocol versão 3 — protocolo para recuperação de emails a partir de um servidor. Permite que clientes façam download de mensagens, geralmente removendo-as do servidor após a transferência.

Porta padrão: 110, 995

Função: Recebimento de emails

Ver guia completo →
Protocolo

IMAP

Internet Message Access Protocol — protocolo moderno para acesso a emails no servidor. Mantém as mensagens no servidor, permitindo sincronização entre múltiplos dispositivos e gerenciamento de pastas.

Porta padrão: 143, 993

Função: Acesso remoto a emails

Ver guia completo →
Protocolo

HTTP & HTTPS

HyperText Transfer Protocol — protocolo utilizado para transferência de dados na Web. O HTTPS é a versão segura com criptografia SSL/TLS. Essenciais para navegação e acesso a serviços web.

Porta padrão: 80 (HTTP), 443 (HTTPS)

Função: Transferência de dados web

Ver guia completo →
Protocolo

DNS

Domain Name System — protocolo responsável por traduzir nomes de domínio em endereços IP. Essencial para que os usuários acessem websites digitando nomes em vez de números IP complexos.

Porta padrão: 53

Função: Resolução de nomes

Ver guia completo →
Protocolo

FTP

File Transfer Protocol — protocolo para transferência de arquivos entre computadores em rede. SFTP oferece a mesma funcionalidade com criptografia SSH integrada.

Porta padrão: 21 (FTP), 22 (SFTP)

Função: Transferência de arquivos

Ver guia completo →
Protocolo

IPv4 & IPv6

Protocolos de endereçamento de rede. IPv4 usa 32 bits (4,3 bilhões de endereços), enquanto IPv6 usa 128 bits com endereços ilimitados para o futuro.

Notação: IPv4: 192.168.1.1, IPv6: 2001:db8::1

Função: Endereçamento de rede
